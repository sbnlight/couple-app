# IDEAS.md — 「我们」页功能备选清单

> 2026-06-10 讨论产出的待选功能,按推荐优先级分梯队。
> 状态:[ ] 未开始 / [x] 已完成。要做哪个时在对话里点名即可。
> 前提:主线里程碑 M2(聊天)、M3(记账)优先于本清单。

## 第一梯队:异地恋刚需,成本低收益高

- [ ] **对方的当地时间**
  名片卡显示「TA 那边现在 21:34 🌙」。两人分处中国/美国,时差 12–15 小时,
  发消息前看一眼对方是否深夜。实现:profiles 加时区字段(或本地配置),纯前端渲染。

- [ ] **下次见面倒数日**
  「距离下次见面还有 47 天 ✈️」。设一个日期即可,存 couples 表加列或单独小表。
  见面日期过期后可归档/设下一次。

- [ ] **正式纪念日**
  目前「小屋已建立 N 天」用的是建小屋日期;改为用户自设的「在一起」纪念日,
  支持多个纪念日(周年、生日)。M4 做了 Web Push 后可做提醒。

## 第二梯队:增加日常互动感

- [ ] **每日一问**
  每天出一个问题,双方各自回答后才能互看答案。题库内置(几百条),
  表:daily_questions(题库)+ answers(couple_id, question_id, user_id, content)。

- [ ] **打卡 / 想你按钮**
  一键「今天想你了 💭」记录连续天数;或自定义共同习惯打卡(早睡/喝水/健身)。

- [ ] **愿望清单**
  共同 bucket list,完成一起打勾。CLAUDE.md 预留方向,表结构与 expenses 同构,
  实现成本低。

## 第三梯队:好但有代价

- [ ] **照片墙 / 共同相册**
  温馨但吃存储(免费额度 1GB),需严格压缩 + 数量预期管理,建议靠后做。

- [ ] **留言小纸条**
  「慢消息」:写一张纸条,对方第二天(或指定时间)才能打开。仪式感强,锦上添花。

## 设置类小项

- [ ] **修改密码**
  「我们」页直接改密码入口(supabase.auth.updateUser),不必走忘记密码收邮件。

- [ ] **检查更新**
  手动刷新到最新版本的按钮(触发 service worker 更新 + reload),
  免去「完全关掉重开」的口头教学。
